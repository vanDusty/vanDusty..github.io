Mybatis 配置多数据源最简解决方案

发表于 2019-05-05 分类于 Mybatis
本文字数： 6.2k 阅读时长 ≈ 11 分钟

现在主流的多数据源切换方式有：AOP 动态切换来做多数据源或者升级成Mybatis-Plus可以简单的配置多数据源，偶然见到大佬整理的很简单的多数据源的配置，改进一下，分享出来。

二、数据源配置

2.1 主数据源配置

  1. 首先创建 DataSource，然后创建 SqlSessionFactory，再创建事务，最后包装到 SqlSessionTemplate 中；
  2. @MapperScan(basePackages = "***", sqlSessionTemplateRef = "***"):指定主库扫描的 mapper 文件，并注入指定的 SqlSessionTemplate,同时SqlSessionFactory的bean中指定xml文件位置;
  3. @Primary 说明指定了主库。

2.2 从库数据源配置

相对于主库的配置，不同点有：

  1. 方法上没有@Primary的注解，说明是从库，(如果需要扩展，按照次配置新增即可)；
  2. @MapperScan(basePackages = "***", sqlSessionTemplateRef = "***"):指定主库扫描的 mapper包 不同；
  3. @ConfigurationProperties(prefix = "spring.datasource.druid.**")：指定的数据源不同。
